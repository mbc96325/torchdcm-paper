# Model-Family Benchmark Comparison

This note groups the remote benchmark results by model family for the IJOC
software-paper experiments. The main paper-facing runtime table reports total
runtime and a single consistency flag only. A separate case-scale table records
sample size `N`, parameter count `K`, and the alignment mode. Estimation/covariance
splits remain in generated validation artifacts.
The public-facing tables use a single `Consistent?` flag. A full-estimation
case is marked `Yes` when the aligned reference comparison satisfies the
paper's prespecified tolerances for likelihood, parameters, probabilities,
and standard errors where directly comparable. Raw difference audits remain
in the generated validation artifacts.

## Solver Coverage Audit

The full solver-attempt matrix is generated by
`validation/benchmarks/run_solver_attempt_matrix.py`. It attempts every
benchmark case against every configured solver column and records one of three
states: `ok` for an aligned completed run, `failed` for an aligned wrapper that
was attempted but did not complete, and `unsupported` for solver/model/dataset
combinations where no aligned wrapper currently exists.

Current full-profile coverage:

| Solver | ok | failed | unsupported |
| --- | ---: | ---: | ---: |
| TorchDCM | 23 | 0 | 0 |
| SciPy BFGS | 1 | 0 | 22 |
| Biogeme | 23 | 0 | 0 |
| Apollo | 20 | 0 | 3 |
| mlogit | 11 | 0 | 12 |
| gmnl | 1 | 1 | 21 |
| xlogit | 1 | 1 | 21 |

Biogeme is now run for every case in the matrix. The additional mlogit-package
datasets use a generic MNL adapter that converts the aligned long-format
TorchDCM specification to Biogeme wide format while preserving the same
alternative-specific variables, availability, zero initial values, and no-ASC
or ASC structure used by the TorchDCM case.

Apollo is now included for all aligned MNL cases in the matrix, including NHTS
2022, the Biogeme public MNL cases, the mlogit-package cases, and the ragged
ModeCanada choice sets through an availability-aware wide-format adapter. The
remaining Apollo `unsupported` cells are Optima ordered logit/probit and
Swissmetro cross-nested full estimation, which need dedicated Apollo wrappers
rather than the generic MNL adapter.

The two current attempted failures are both on ModeCanada: `gmnl` reports a
non-conformable-array error, and `xlogit` rejects the ragged long-format choice
sets. These are kept as `failed` rather than converted to `unsupported` because
the wrappers were actually run. Paper-facing runtime tables should continue to
include only aligned comparable cells; the solver-attempt matrix is the audit
trail showing that missing cells were checked rather than silently omitted.

## Reading The Table

- `full_estimation` means every available backend estimates parameters on
  the aligned public dataset.
- `torch_fit_then_replay` means TorchDCM estimates parameters and reference
  packages replay the same fitted parameters to verify likelihood and
  probability kernels. These rows are not included in the main paper-facing
  Table 1.
- `fixed_replay_shared_draws` means all backends use identical fixed
  parameters and simulation draws. This validates mixed-logit probability
  and log-likelihood kernels, but it is not yet a full estimation benchmark.

## Current Interpretation

- MNL is the strongest completed family: Swissmetro plus public Biogeme
  data and R community data are consistent under the paper tolerance rule.
- Nested logit full estimation is consistent with Biogeme/Apollo. Biogeme's
  nested covariance parameterization should still be audited before the
  manuscript table is frozen.
- Cross-nested logit now has full-estimation comparison against Biogeme.
  It is consistent under the paper tolerance rule, with constrained
  covariance interpretation still worth documenting.
- Mixed logit and WTP-space mixed logit have shared-draw fixed replay
  parity against Biogeme and Apollo. The next required step is full
  simulated maximum-likelihood estimation with shared random streams or
  deterministic draws.
- Ordered logit/probit full estimation on Optima is consistent with Biogeme.
- NHTS 2022 is now included as an aligned public travel-survey MNL benchmark
  using a deliberately simple trip-mode specification. NHTS 2017, UK NTS, MiD,
  PSRC, and other survey sources remain pending until processed choice sets and
  estimator-identical specifications are available.
- Additional collected datasets are added only after an aligned specification
  is available. Cases such as Car, Game/Game2, JapaneseFDI, and large raw travel
  surveys without a processed choice specification are intentionally excluded
  from the paper-facing table rather than forced into a generic benchmark.

## Next Benchmark Work

1. Add full mixed-logit estimation on Swissmetro with deterministic
   antithetic draws and compare against Biogeme/Apollo.
2. Add mixed-logit comparisons with `gmnl` and `xlogit` on Fishing or
   Electricity once the data shape and draw conventions are aligned.
3. Extend nested-logit validation beyond Swissmetro using another public
   Biogeme/Apollo example with a nontrivial nest structure.
4. Decide how manuscript tables should report covariance for constrained
   nest parameters, because that is currently the largest non-runtime
   discrepancy.
